SFGraphics.GLObjects Namespace

 
Classes
  ClassDescription
BufferObject
Encapsulates an OpenGL buffer object. Like other GLOBjects, memory is handled by GLObjectManager.
Framebuffer
Encapsulates an OpenGL framebuffer, including any attached color or depth attachments.
GLObjectManager
Manages reference counting and resource management for all GLObjects. Each GLObject has an integer ID generated by OpenTK. When an object is created, the reference count for that ID is incremented. When the finalizer is called for an object, the reference count is decremented.

GLObjectManager will maintain reference counts for Shader or any class inheriting from Texture, BufferObject, or Framebuffer.

Call DeleteUnusedGLObjects with a valid OpenTK context current to delete objects with no references. The reference count is managed automatically, so there is no need to call GL.DeleteTexture(int), for example.

Renderbuffer
Encapsulates a renderbuffer object, which are used exclusively as attachments for Framebuffer objects. Renderbuffers support multisampling.
