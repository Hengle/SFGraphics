GLObjectManager Class

Manages reference counting and resource management for all GLObjects. Each GLObject has an ID generated by OpenTK. When an object is created, the reference count for that ID is incremented. When the finalizer is called for an object, the reference count is decremented. GLObjectManager will maintain reference counts for any class inheriting from Texture or BufferObject.

Call DeleteUnusedGLObjects with a valid OpenTK context current to delete objects with no references. The reference count is managed automatically, so there is no need to call GL.DeleteTexture(int), for example.

DeleteUnusedGLObjects
The appropriate GL.Delete() function is called for all GLObjects if the ID has 0 references. This means GLObjects may not be cleaned up until long after the original object becomes unreachable.

Call WaitForPendingFinalizers prior to DeleteUnusedGLObjects if more immediate cleanup is desired.
